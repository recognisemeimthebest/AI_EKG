# CN-0002: AI 기능 구성 재정의 (2종 -> 3종)

- 작성일: 2026-04-06
- 영향 범위: AI/ML, 소프트웨어, 기획 전체

## 결정 사항

AI EKG 모니터의 핵심 AI 기능을 기존 2종에서 3종으로 재정의한다.

| # | 기능 | 상태 | 비고 |
|---|------|------|------|
| 1 | 실시간 부정맥 분류 (CNN-TCN 3-lead, 90.6%) | **완료** | 기존 유지 |
| 2 | 발작성 AF 감지 (정상 리듬 ECG에서 숨겨진 AF 흔적) | **신규** | 우선순위 1 |
| 3 | AFib 예측 (시퀀스 TCN + 임상정보 결합) | **개선** | 우선순위 2 |

## 배경/맥락

- ML 모델 2종 개발 완료 후, 연구 논문 조사를 통해 발작성 AF 감지의 임상적 가치와 기술적 타당성을 확인
- 부정맥 분류(모델 1)는 "현재 상태"를 진단하지만, 발작성 AF는 "겉으로 정상인데 실제로는 AF 위험이 있는 환자"를 스크리닝하는 별개의 가치
- AFib 예측(모델 3)은 현재 AUROC 0.7355로 임상 활용에 한계가 있어 개선 필요

## 검토한 대안

### 대안 A: 기존 2종 유지, 소프트웨어 개발 우선
- 장점: 빠른 완제품 도달
- 단점: 예측 모델 AUROC 0.74는 실용성 의문, 제품 차별화 부족
- 기각 이유: 하드웨어 대기 시간 활용 가능, 모델 추가가 비용 효율적

### 대안 B: 발작성 AF 감지만 추가 (3종, 예측 모델 개선 안 함)
- 장점: 단순, 빠름
- 단점: 예측 모델이 AUROC 0.74로 남아 실용성 부족
- 기각 이유: 임상정보 결합은 데이터만 추출하면 구현 가능, 비용 대비 효과 높음

### 대안 C (채택): 3종 구성 (분류 + 발작성 AF + 예측 개선)
- 장점: 3가지 다른 관점의 AF 분석 (현재 상태 / 숨겨진 흔적 / 미래 예측), 논문 기반 성능 기대치 높음
- 단점: 개발 기간 추가 (2~4주)
- 채택 이유: 하드웨어 대기 기간 활용 가능, CNN-TCN 백본 재활용으로 효율적

## 근거

### 발작성 AF 감지
- Attia et al. (Mayo Clinic, 2019): 정상 리듬 12-lead ECG에서 AF 감지 AUROC 0.87
- 3-lead에서도 0.83~0.85 수준 기대 (리드 수 감소에 의한 성능 저하 ~3%p)
- 기존 CNN-TCN 백본 활용으로 Transfer Learning 가능 -> 학습 비용 낮음

### AFib 예측 임상정보 결합
- Tarabanis et al. (EHJ Digital Health, 2025): ECG + CHARGE-AF 11개 변수 = AUC 0.89
- 가장 효과적 변수: 병력(PMH) - 당뇨, 심근경색, 심부전, 항고혈압제
- 핵심 차이: 기존 HRV/ECG Intervals 실험은 CNN이 이미 학습한 정보와 중복이라 실패했지만, 임상 병력은 ECG 파형에 존재하지 않는 외부 정보
- MIMIC-IV diagnoses_icd 테이블에서 ICD 코드로 직접 추출 가능

### 우선순위 결정
- 발작성 AF 감지 먼저: 데이터 구축이 상대적으로 단순 (AF 진단 + sinus rhythm 필터링), 기존 백본 재활용
- AFib 예측 개선 다음: diagnoses_icd 파싱 및 환자 매칭 로직이 더 복잡

## 영향/주의사항

1. **Phase 구조 변경**: Phase 4를 ML 모델 개발로, Phase 5를 소프트웨어 개발로 조정
2. **추론 서버 부하**: 모델 3종 순차 추론 시 지연 시간 검증 필요 (Phase 5 Gate)
3. **UI 설계 변화**: 5인치 LCD에 3가지 AI 결과를 어떻게 표시할지 UX 설계 필요
4. **규제 포지셔닝**: 발작성 AF 감지는 Apple Watch 수준의 "추가 검사 권장" 안내로 제한
5. **데이터 주의**: 발작성 AF 감지 데이터 구축 시, AF 진단 환자의 "정상 리듬 ECG"를 정확히 필터링해야 함 (rhythm label 확인 필수)
