# AI EKG 모델 개발 여정 — 타협과 선택의 기록

> 3개의 AI 모델을 만들면서 겪은 시행착오, 실패, 그리고 타협의 과정을 일자별로 정리한 기록입니다.
> 최종 결과만 보면 깔끔해 보이지만, 실제로는 수많은 실패와 방향 전환이 있었습니다.

---

## 프로젝트 배경

ESP32 + ADS1292R 기반 3-lead EKG 측정 장치에 탑재할 AI 모델 3종을 개발하는 과정입니다.
학습 데이터는 MIMIC-IV ECG (775,367건, 159,666명 환자)를 사용했습니다.

**최종 결과 요약:**

| 모델 | 용도 | 성능 | 파라미터 |
|------|------|------|----------|
| CNN-TCN 3-lead | 실시간 부정맥 분류 | 90.6% Accuracy | 126K |
| ResNet-34 + Clinical | 발작성 AF 감지 | AUROC 0.8240 | 7.3M |
| Sequence TCN | AFib 예측 (15일) | AUROC 0.7355 | 145K |

여기까지 오는 데 아래와 같은 여정이 있었습니다.

## Phase 1: 데이터 준비 (2026-03-29 ~ 04-01)

### Day 1 (03-29): 프로젝트 킥오프

처음 목표는 단순했습니다 — "부정맥 분류 모델 하나 만들자."

MIMIC-IV ECG 데이터 80만건이 있었고, 12-lead ECG를 3-class(Normal/AFib/Other)로 분류하는 모델을 만들기로 했습니다. 하지만 첫 번째 문제는 데이터 전처리였습니다.

- 80만건의 WFDB 파일 (.dat/.hea)을 하나의 HDF5로 합쳐야 했음
- report_0 텍스트 라벨이 1,570종 — 이걸 3개 클래스로 매핑해야 했음
- "sinus rhythm." "Sinus rhythm" "SINUS RHYTHM" 같은 변형이 수십 가지

**타협 1: 라벨 매핑 전략**

처음엔 정규표현식으로 정밀하게 매핑하려 했으나, 변형이 너무 많았습니다.
결국 소문자 변환 + 키워드 포함 여부로 단순화했습니다. 정밀도를 약간 포기하고 커버리지를 택한 셈입니다.

### Day 2-3 (03-30 ~ 04-01): 전처리 파이프라인

- Bandpass filter (0.5-50Hz) + 60Hz Notch filter + Z-score 정규화
- 처리 속도: ~66건/초 → 전체 78만건 처리에 약 3.3시간
- 출력 파일 크기: **173GB** (gzip 압축 포함)

**타협 2: gzip 압축이 병목**

전처리에서 가장 느린 부분은 신호 처리가 아니라 gzip 압축이었습니다. 압축 없이 저장하면 300GB+가 되어 디스크가 부족하고, 압축하면 느렸습니다. gzip level 4로 타협했습니다.

**타협 3: 12-lead 전체 저장 vs 3-lead만 저장**

최종 기기는 3-lead(I, II, III)만 사용하지만, 12-lead 전체를 저장했습니다. 이유는:
- 12-lead 성능을 기준선(baseline)으로 알고 싶었음
- 나중에 다른 실험에도 재활용 가능
- 디스크 용량이 아슬아슬하게 충분했음 (G드라이브 SSD 여유 공간 확인 후 결정)

## Phase 2: 부정맥 분류 모델 (2026-04-02 ~ 04-05)

### Day 4 (04-02): 첫 번째 모델 — CNN-TCN 12-lead

CNN 3블록 + TCN 4레이어 구조를 설계했습니다. 126,355 파라미터의 가벼운 모델입니다.
(라즈베리파이 5에서 추론해야 하니까 모델을 작게 유지해야 했습니다.)

첫 학습 결과:

| 실험 | 입력 | Accuracy |
|------|------|:--------:|
| CNN-TCN 12-lead | 12리드 전체 | **93.6%** |

93.6%! 좋은 출발이었지만, 현실의 벽이 기다리고 있었습니다.

### Day 5 (04-03): 현실의 벽 — 12-lead vs 3-lead

우리 기기는 3-lead(I, II, III)만 측정합니다. 12-lead 모델은 실제로 쓸 수 없습니다.

**타협 4: 12-lead 성능(93.6%)을 포기하고 3-lead로 전환**

이것이 프로젝트 전체에서 가장 큰 타협이었습니다. V1~V6 흉부유도 없이 사지유도 3개로만 해야 합니다. 성능 하락은 불가피했습니다.

### Day 6 (04-04): 실패한 시도들

성능을 보상하기 위해 두 가지 시도를 했습니다. 둘 다 실패했습니다.

**실패 1: 혈압(BP) 피처 추가 → Accuracy 90.3%**

ICU chartevents + OMR에서 혈압을 추출해 보조 피처로 넣었습니다. 하지만 BP 매칭률이 29.9%에 불과해 70%가 결측값이었고, 이 결측 노이즈가 오히려 성능을 깎았습니다.

- 교훈: 매칭률이 낮은 피처를 억지로 넣으면 역효과

**실패 2: CBAM 어텐션 추가 → Accuracy 90.6%**

"어텐션 메커니즘을 넣으면 좋아지겠지?"라는 기대로 CBAM(Channel & Spatial Attention)을 추가했습니다. 결과는 12-lead에서 90.6%로 오히려 하락.

- 교훈: 126K 파라미터 규모의 소규모 모델에서 어텐션은 파라미터 낭비
- 어텐션이 효과를 발휘하려면 모델이 충분히 크고 학습할 정보가 풍부해야 함

### Day 7 (04-05): 타협의 결과 수용

결국 가장 단순한 방법이 답이었습니다:

| 실험 | 입력 | Accuracy | 결과 |
|------|------|:--------:|------|
| CNN-TCN 12-lead | 12리드 | 93.6% | 기기에서 사용 불가 |
| CNN-TCN 12-lead + BP | 12리드 + 혈압 | 90.3% | 결측 노이즈 |
| CNN-TCN-CBAM 12-lead | 12리드 + 어텐션 | 90.6% | 어텐션 비효율 |
| **CNN-TCN 3-lead** | **3리드** | **90.6%** | **최종 채택** |

**타협의 결론:**
- 12-lead(93.6%) → 3-lead(90.6%)로 **3%p 하락 수용**
- BP 추가, 어텐션 추가 모두 실패 → 단순한 구조 + 3-lead 그대로 사용
- "파형 자체에 대부분의 정보가 있다" — 이후 실험에서 반복적으로 확인되는 교훈

## Phase 3: AFib 예측 모델 (2026-04-05 ~ 04-06)

### Day 7-8 (04-05 ~ 04-06): "미래를 예측할 수 있을까?"

부정맥 분류는 "지금 이 ECG가 뭔지" 알려줍니다. 하지만 더 가치 있는 질문은 "앞으로 부정맥이 올까?"입니다.

아이디어: 같은 환자의 연속 ECG 기록(수일~수주에 걸쳐 여러 장)을 시계열로 보고, 15일 내 리듬 이상 발생을 예측하자.

**7번의 실험, 7번의 타협:**

| # | 시도 | AUROC | 왜 안 됐나 |
|---|------|:-----:|-----------|
| 1 | 30일 윈도우, backbone freeze | 0.705 | 2,770 params만 학습 → 표현력 부족 |
| 2 | 30일 윈도우, 전체 finetune | 0.705 | 차이 없음 → 30일이 너무 넓은 윈도우 |
| 3 | 15일 윈도우로 축소 | 0.7185 | 약간 개선 → 윈도우 축소 효과 확인 |
| 4 | + HRV 피처 추가 | 0.7073 | **오히려 하락** → CNN이 이미 학습한 정보 |
| 5 | + ECG Intervals 추가 | 0.7114 | **오히려 하락** → 역시 중복 정보 |
| 6 | 리듬이상 그룹 통합 | 0.7055 | 양성 비율 증가했지만 AUROC 소폭 하락 |
| 7 | **시퀀스 TCN** | **0.7355** | **최종 채택** |

**핵심 교훈: "피처 추가는 만능이 아니다"**

HRV(심박변이도), ECG Intervals(P파/QRS/QTc 지속시간)을 추가하면 당연히 좋아질 거라 기대했습니다. 하지만 **CNN이 이미 파형에서 같은 정보를 학습하고 있었기 때문에**, 중복 피처는 효과가 없었고 오히려 노이즈가 되었습니다.

이 실패에서 얻은 인사이트:
1. CNN이 파형에서 추출하는 것과 **다른 종류의 정보**만 효과가 있음
2. 단일 ECG(0.72)의 한계를 넘으려면 **연속 ECG의 변화 패턴**을 봐야 함
3. AFib만 보는 것보다 **유사 질환(AFlutter, SVT 등)을 묶으면** 양성 샘플이 늘어 학습 효율 상승

**타협 5: AUROC 0.74를 일단 수용**

목표는 0.80이었지만 0.7355에 머물렀습니다. 하지만 이 모델에는 다른 강점이 있었습니다:
- 임계값 0.5에서 Precision 38%, Recall 26%
- **15일 연속 측정 시 누적 Recall: 1-(0.74)^15 ≈ 98%**
- 매일 측정하는 기기 특성상, 단일 측정 성능보다 누적 감지율이 중요

"한 번에 못 잡아도 매일 보면 결국 잡는다" — 이 기기의 사용 시나리오에 맞는 타협이었습니다.

## Phase 4: 발작성 AF 감지 모델 (2026-04-06 ~ 04-07)

### Day 9 (04-06): 방향 전환 — "세 번째 모델이 필요하다"

예측 모델(AUROC 0.74) 결과에 만족하지 못하던 중, 논문 조사를 통해 새로운 아이디어를 발견했습니다.

> "정상 리듬으로 보이는 ECG에서도 AF의 흔적을 감지할 수 있다" — Attia et al., Mayo Clinic, 2019

이것은 부정맥 분류(지금 AFib인가?)와도, AFib 예측(앞으로 올까?)와도 다른 질문입니다:
- **"지금 정상으로 보이지만, 사실 이 사람은 AF 환자인가?"**

임상적으로 매우 가치 있는 기능이었습니다. 발작성 AF 환자는 증상이 없을 때 ECG를 찍으면 정상으로 나오기 때문에 진단이 어렵습니다.

**타협 6: 2종 체제에서 3종 체제로 전환**

원래 계획은 모델 2종(분류 + 예측)이었습니다. 3종으로 늘리면 개발 시간이 추가되지만:
- 하드웨어 부품이 아직 배송 중이라 대기 시간 활용 가능
- 제품 차별화에 큰 기여
- 기존 데이터를 재가공해서 쓸 수 있음

### Day 9 (04-06): 첫 시도 — Transfer Learning으로 빠르게

"기존 CNN-TCN 백본을 재활용하면 빠르게 만들 수 있겠다."

부정맥 분류용 CNN-TCN(126K params)의 backbone을 freeze하고, 이진분류 헤드만 교체해서 학습.

**결과: AUROC 0.68 — 완전한 실패**

왜 실패했는가:
- 부정맥 분류(Normal/AFib/Other 구분)와 발작성 AF 감지(정상 리듬 내 미세한 AF 흔적)는 **근본적으로 다른 피처**를 학습해야 함
- 부정맥 분류 모델은 "지금 AFib인지"를 보지만, 발작성 AF 감지는 "정상으로 보이는데 숨겨진 구조적 변화"를 봐야 함
- 126K의 작은 모델로는 이런 미세한 패턴을 포착할 수 없음

**타협 7: Transfer Learning 포기, from scratch로 전환**

빠르게 만들겠다는 계획을 접고, 처음부터 더 큰 모델을 학습하기로 했습니다.

### Day 10 (04-07): ResNet-34 — 큰 모델이 답이었다

참고 논문(Tarabanis et al., 2025)의 방법론을 따라 ResNet-34(7.3M params)를 from scratch로 학습.

| 단계 | 모델 | AUROC | 교훈 |
|------|------|:-----:|------|
| 1 | CNN-TCN Transfer Learning | 0.68 | 다른 과제엔 다른 모델 필요 |
| 2 | ResNet-34 ECG only | 0.8038 | 큰 모델 + 52만건 데이터 = 효과적 |
| 3 | **ResNet-34 + 임상정보** | **0.8240** | **최종 채택** |

**임상정보 추가가 효과 있었던 이유 (이전 실패와의 차이):**

Phase 3에서 HRV/ECG Intervals 추가가 실패한 이유는 CNN이 이미 파형에서 학습한 정보와 중복이었기 때문입니다. 하지만 임상정보(당뇨, 심부전, 심근경색, 항고혈압제)는 **ECG 파형에 존재하지 않는 외부 정보**입니다.

- HRV/ECG Intervals: 파형에서 계산 가능 → CNN과 중복 → 효과 없음
- 임상 병력: 파형에 없는 정보 → CNN을 보완 → **+0.02 AUROC 향상**

**타협 8: AUROC 0.82 수용 (목표 0.83 미달)**

논문은 8-lead + CHARGE-AF 11개 변수로 0.89를 달성했습니다. 우리는 3-lead + 4개 변수로 0.82.

| | 논문 | 본 프로젝트 | 차이 원인 |
|---|:---:|:---:|------|
| 리드 수 | 8-lead | 3-lead | V1 없음 (AF P파 관찰에 중요) |
| 임상 변수 | 11개 (CHARGE-AF) | 4개 | BMI, 인종, 흡연 등 미포함 |
| AUROC | 0.89 | 0.82 | -0.07 |

하지만 논문의 ECG only 결과(0.83)와 우리의 0.82는 거의 동등합니다.
**V1 리드 없이 Lead II의 P파 형태만으로 AF 흔적을 학습한 것**은 주목할 만한 결과입니다.

### 임계값 최적화 — 마지막 타협

모델 성능은 고정이지만, 임계값 조정으로 실용성을 크게 바꿀 수 있었습니다:

| 전략 | 임계값 | Sensitivity | Specificity | Precision |
|------|:---:|:---:|:---:|:---:|
| 기본값 | 0.50 | 71% | 77% | 25% |
| **권장 설정** | **0.61** | **60%** | **85%** | **31%** |
| 보수적 | 0.78 | 37% | 95% | 43% |

**타협 9: Sensitivity 71% → 60%로 낮추고, 오탐률(15%)을 관리 가능 수준으로**

임계값 0.5에서는 100명 중 25명만 진짜 양성(Precision 25%) → 알람 피로가 심함.
임계값 0.61에서는 Precision 31%로 개선되고, 반복 측정(매일 10초)으로 누적 감지율 보완 가능.

## 타협의 전체 지도

```
12-lead 93.6% ──── 3-lead 제약 ────→ 90.6% (분류)
    │
    ├── BP 추가 시도 ────→ 실패 (결측 70%)
    ├── CBAM 추가 시도 ──→ 실패 (소규모 모델에 비효율)
    └── 단순한 게 답 ───→ 3-lead CNN-TCN 채택

AUROC 0.87 (논문) ── 3-lead + 적은 변수 ──→ 0.82 (발작성 AF)
    │
    ├── TL 시도 ─────────→ 실패 (0.68, 다른 과제)
    ├── ECG only ────────→ 0.80 (from scratch)
    └── + 임상정보 ──────→ 0.82 (외부 정보 효과 확인)

AUROC 0.80 (목표) ── 단일 ECG 한계 ──→ 0.74 (예측)
    │
    ├── HRV 추가 ────────→ 실패 (CNN 중복)
    ├── Intervals 추가 ──→ 실패 (CNN 중복)
    └── 시퀀스 TCN ───────→ 0.74 (누적 Recall 98%로 보완)
```

## 되돌아보며: 이 프로젝트에서 배운 것

### 1. 단순한 구조가 이긴다 (소규모 모델에서)
126K 파라미터 모델에 어텐션, 추가 피처, Transfer Learning을 시도했지만 대부분 실패. 모델 규모에 맞는 복잡도가 있습니다.

### 2. "다른 종류"의 정보만 효과가 있다
HRV, ECG Intervals → CNN이 이미 파형에서 학습한 것과 중복 → 효과 없음.
임상 병력(당뇨, 심부전 등) → 파형에 없는 정보 → 효과 있음 (+0.02).
피처를 추가하기 전에 "이 정보가 모델이 이미 알고 있는 것과 다른가?"를 물어야 합니다.

### 3. 과제가 다르면 모델도 달라야 한다
부정맥 분류(Normal vs AFib)와 발작성 AF 감지(정상 리듬 내 미세 흔적)는 같은 ECG를 봐도 다른 피처를 필요로 합니다. TL이 항상 답은 아닙니다.

### 4. 기기의 사용 시나리오가 모델의 약점을 보완한다
단일 측정 Recall 26%는 낮아 보이지만, 매일 측정하면 15일 내 누적 Recall 98%.
"모델 성능"보다 "제품으로서의 성능"을 봐야 합니다.

### 5. 3-lead의 한계는 명확하지만, 생각보다 많은 것이 가능하다
- 부정맥 분류: 12-lead 93.6% → 3-lead 90.6% (3%p 손실)
- 발작성 AF: 8-lead 0.89 → 3-lead 0.82 (0.07 손실)
- Lead II의 P파 형태만으로도 상당 수준의 AF 분석이 가능
