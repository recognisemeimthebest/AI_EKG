- 작성일: 2026-03-29
- 최종 수정일: 2026-04-07

## 1. 요약 (Executive Summary)

본 프로젝트는 AI가 탑재된 병실용 EKG(심전도) 측정 장치를 하드웨어부터 소프트웨어까지 전체적으로 개발하는 개인 프로젝트이다. 이전에 개발한 전자청진기(ESP32 + BLE, 95% 완성)의 심음(PCG) 데이터와 EKG 데이터를 결합하여, 환자의 미래 심질환을 예측할 수 있는 융합 시스템을 구축한다.

### 핵심 목표
- **3리드 EKG 측정 장치** 개발 (개인 모니터링 + 병실 베드사이드 겸용)
- **AI 기반 3가지 핵심 분석 기능**:
  - (1) 실시간 부정맥 분류 (Normal/AFib/Other) -- **완료, 90.6%**
  - (2) 발작성 AF 감지 (정상 리듬 ECG에서 숨겨진 AF 흔적 스크리닝) -- **완료, AUROC 0.8240**
  - (3) AFib 예측 (연속 ECG 시퀀스 + 임상정보 기반 15일 내 리듬 이상 예측) -- **개선 예정**
- 전자청진기 PCG + EKG 데이터 융합 (향후 데이터 축적 후)
- 5인치 내장 디스플레이 실시간 심전도 파형 표시
- 위험도 대시보드 (AF 감지 확률 추이, 측정 이력, PDF 리포트)

### 제품 특성
- **사용 환경**: 개인 모니터링 (매일 아침 10초 측정) + 병실 베드사이드 (연속 모니터링)
- **전원**: 상시 5V USB 전원 + 이동용 리튬 배터리
- **통신**: BLE → Raspberry Pi 5 (AI 추론 서버)
- **데이터**: PostgreSQL DB 기록, 측정 이력 저장 (날짜별 ECG + AI 분석 결과)
- **규제 포지셔닝**: 웰니스 기기 (Apple Watch, KardiaMobile 수준). 구체적 치료 지시 불가, "병원 방문 권장" 수준의 일반 안내만 제공
- **인증**: 프로토타입/연구 목적 (의료기기 인증 미고려)

#### JSF++ 채택 원칙 (펌웨어 코딩 표준)
1. 동적 메모리 할당 최소화 — 직접 작성 코드에서 new/malloc 금지, 정적 할당 사용
2. 모든 변수 초기화 필수
3. 재귀 호출 금지
4. 모든 switch에 default 케이스
5. 함수 하나당 하나의 종료점(return)
6. 매직 넘버 금지 — 상수는 반드시 const/define으로 정의
7. 워치독 타이머 활용 — 펌웨어 행(hang) 시 자동 리셋
8. 에러 상태 명시적 처리 — 센서 통신 실패 시 안전한 상태로 전환

### Phase 5: AI/ML 모델 개발
**목표**: 3가지 핵심 AI 기능 완성 (부정맥 분류 + 발작성 AF 감지 + AFib 예측)

| 순서 | 항목 | 상태 | 산출물 |
|------|------|------|--------|
| 1 | PhysioNet/MIMIC-IV 데이터 수집 및 전처리 | **완료** | 775,367건 HDF5, 173GB |
| 2 | **부정맥 분류 모델** (CNN-TCN 3-lead) | **완료** | 90.6% accuracy, 126K params |
| 3 | **발작성 AF 감지 모델** (ResNet-34 + Clinical) | **완료** | AUROC 0.8240, 7.3M params |
| 4 | **AFib 예측 모델 개선** (임상정보 결합) | 예정 | 시퀀스 TCN + CHARGE-AF 변수, AUROC 0.80+ 목표 |
| 5 | 라즈베리파이 배포 최적화 | 예정 | 모델 3종 ONNX 변환/경량화 |
| (보류) | PCG + EKG 융합 모델 | 보류 | 자체 데이터 축적 후 추후 진행 |
| **검증** | **실제 데이터 정확도 테스트** | | **PhysioNet 학습 모델이 실측 데이터에서도 정확한지 확인** |

#### 5-1. 실시간 부정맥 분류 (완료)
- **모델**: CNN 3블록 + TCN 4레이어 + FC
- **입력**: 3-lead ECG (10초, 500Hz) + numeric(4) + patient(2)
- **출력**: 3클래스 (Normal / AFib / Other)
- **성능**: Test Accuracy 90.6%, 126,355 params
- **체크포인트**: `checkpoints/cnn-tcn-3lead/best_model.pt`
- **용도**: 연속 모니터링 중 부정맥 발생 시 즉시 분류 + "병원 방문 권장" 수준 안내

#### 5-2. 발작성 AF 감지 (완료, AUROC 0.8240)
- **목적**: 정상 리듬 ECG 1장에서 숨겨진 AF 흔적 감지
- **방식**: AF 진단 환자(+-90일 내)의 정상 리듬 ECG vs AF 이력 없는 환자 ECG 이진분류
- **아키텍처**: ResNet-34 (1D Conv, from scratch) + Late Fusion (임상정보 4종)
  - CNN-TCN Transfer Learning은 AUROC 0.68 정체로 기각
- **데이터**: 52만건 (양성 52,202 / 음성 471,157, 1:9 비율)
- **성능**: Test AUROC 0.8240, Sensitivity 71%, Specificity 77%
- **권장 임계값**: 0.61 (Sens 60%, Spec 85%, Prec 31%)
- **논문 비교**: 논문 ECG only AUROC 0.83 vs 본 프로젝트 0.82 (3-lead 한계 내 거의 동등)
- **용도**: "정상처럼 보이지만 AF 가능성 있으니 추가 검사 권장" 스크리닝
- **출력 예시**: "AF 흔적 감지 확률: 12%"
- **체크포인트**: `checkpoints/paroxysmal-af-resnet34-clinical/`

#### 5-3. AFib 예측 개선 (우선순위 2)
- **현재 성능**: 시퀀스 TCN, AUROC 0.7355
- **개선 방향**: MIMIC-IV diagnoses_icd에서 임상정보 결합
  - 병력(PMH): 당뇨, 심근경색, 심부전, 항고혈압제 사용 여부
  - 참고 논문: Tarabanis et al. (EHJ Digital Health 2025) -- ECG + CHARGE-AF 11개 변수로 AUC 0.89
- **목표**: AUROC 0.74 -> 0.80+
- **핵심 차이**: 기존 피처 추가 실험(HRV, ECG Intervals)은 CNN이 이미 학습한 정보와 중복이라 실패했지만, 임상 병력(당뇨/심부전 등)은 ECG 파형에 없는 외부 정보이므로 효과 기대

## 4. 시스템 아키텍처

```
[3리드 전극 (Ag/AgCl)]
        │
        ▼
[ADS1292R 모듈] ─── 2채널 측정, Lead III 계산
        │ SPI
        ▼
[ESP32-WROOM-32D]
   │         │
   │         ├── BLE ──────►[Raspberry Pi 5 (8GB)] — AI 추론 서버
   │         │                  ├── BLE 수신 + 데이터 파이프라인
   │         │                  ├── PostgreSQL (측정별 ECG + AI 결과 저장)
   │         │                  ├── AI 모델 3종 추론:
   │         │                  │   ├── (1) 부정맥 분류 (실시간)
   │         │                  │   ├── (2) 발작성 AF 감지 (측정 시)
   │         │                  │   └── (3) AFib 예측 (누적 데이터 기반)
   │         │                  ├── 위험도 대시보드 (AF 확률 추이, 이력)
   │         │                  └── PDF 리포트 생성 (병원 전송용)
   │         │
   └──► [5인치 SPI LCD] 실시간 파형 + 심박수 + AI 결과 표시

[전원: 5V USB + 리튬 배터리 백업]

[전자청진기] ── BLE ──► 핸드폰 ──► 같은 Raspberry Pi 서버
```

### 사용 시나리오 (개인 모니터링)
```
매일 아침 ECG 측정 (10초)
  → 실시간 파형 표시 + 심박수/HRV
  → [모델 1] 부정맥 분류: "정상 리듬입니다" 또는 "심방세동 감지 - 병원 방문 권장"
  → [모델 2] 발작성 AF 감지: "AF 흔적 감지 확률: 12%" (숨겨진 AF 스크리닝)
  → [모델 3] 누적 데이터 기반 AF 예측: "리듬 이상 위험도: 주의" (연속 측정 기반)

부가 기능:
  → 측정 이력 저장 (날짜별 ECG 기록, 트렌드 그래프)
  → 위험도 대시보드 (AF 감지 확률 추이)
  → 병원 전송용 PDF (ECG 파형 + AI 분석 결과)
```

## 7. 전자청진기 데이터 연동 계획

### 7.1 현재 전자청진기 시스템
- MCU: ESP32
- 통신: BLE → 핸드폰 → 서버
- 기능: 심음(PCG) 측정, 정상/잡음 분류, 잡음 종류 구분 (AI 모델)
- 완성도: 약 95%

### 7.2 연동 방식
1. **서버 데이터 병합**: 전자청진기와 EKG 모두 같은 라즈베리파이 서버로 데이터 전송 → 환자 ID 기준 PostgreSQL에서 병합
2. **데이터 포맷 통일**: 공통 포맷 (타임스탬프, 환자ID, 신호값)
3. **독립 동작 우선**: 반드시 동시 사용할 필요 없음, 각각 데이터가 쌓이면 서버에서 분석
4. **동시 사용 시 시너지**: 실시간 PCG + EKG 동기화 분석 가능

### 7.3 융합 AI 모델 (보류)
- PCG + EKG 융합 모델은 **자체 데이터 축적 후** 개발 예정
- 현재는 EKG 단독 분석 모델에 집중
